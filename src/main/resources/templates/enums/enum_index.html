用枚举来维护页面的一些常量，如下拉选框中的选项、状态位显示(如数据库存的编码,但页面要显示其含义)、或部分常量。
用枚举这样设计的好处是有一个地方可以统一管理，有新增只用在枚举类中引入，类似编码含义也可以是通过枚举的定义得到。
但，每次都需要更改代码。所以，更好的当然是数据库来维护，有界面给使用者自己去维护。
(只是自己单纯的一种思路，并不觉得特别理想，希望能接触到更好的此类设计。)

public class SongsBean {
private SongsEnum songsEnum;

}
